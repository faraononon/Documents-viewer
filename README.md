# 📄 Document Viewer with Annotations

Реализация функционала просмотра документов и добавления аннотаций. Архитектура построена следующим образом:

- Основная **entity** — это компонент **`document-viewer`**, и **`pages`** компонент для отображения страниц.
- Далее реализован **feature** слой **аннотаций**, в нем:
  - **поп-ап** аннотации
  - **бэйдж** аннотации
  - **сервис** по взаимодействию с ними
- Выделен **data-access** слой для общения с сервером:
  - **`document-request`** сервис для запросов на бэкенд
  - **модели** dto и models для работы с данными
  - **маппер** `document-mapper` Так же добавил маппер для более удобного взаимодействия с бэкендом в случае расширения модели бэкенда (чтобы не тянуть лишнее на фронт)
- **shared** слой имеет два **root** сервиса:
  - для **zoom** функционала
  - для **сборки модели** документа для отправки
- **assets** слой для различных картинок и т.д (у меня в данном случае это **json** мок)
- **pages** положил картинки из ТЗ чтобы не переписывать мок (так бы тоже ушли в assets)

---

По поводу дальнейшего расширения добавления других типов аннотаций

- на данный момент реализовал через проекцию контента (пример использования ниже).
  Но на данный момент аннотация все равно будет сохранятся в строке. Для более комплексного подхода я бы использовал ControlValueAccessor и закладывал бы фабрику типов для разных контролов (например для фото)

  ```typescript
   <app-annotation-popup>
     <ng-template #content let-annotation="annotation" let-setAnnotation="setAnnotation">
       <input type="text" [ngModel]="annotation" (ngModelChange)="setAnnotation($event)" />
     </ng-template>
   </app-annotation-popup>
  ```

  ***

  Что еще можно улучшить:
  - добавить валидацию для контрола в поп-апе (я сделал ограничение в 200)
  - вынести ui составляющую поп-апа и бэйджа в ui слой, либо внедрить ui-kit
